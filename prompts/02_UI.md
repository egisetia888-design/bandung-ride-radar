02_UI.md

Sprint 2 — UI Foundation

Objective

Membangun fondasi antarmuka Bandung Ride Radar berdasarkan "DESIGN.md".

Sprint ini hanya berfokus pada Design System, layout dasar, dan reusable components.

Jangan mengimplementasikan logika bisnis ataupun integrasi API.

---

Context

Sebelum memulai, baca kembali:

- docs/PRD.md
- docs/DESIGN.md
- docs/CODING_GUIDE.md
- docs/ARCHITECTURE.md

Ikuti seluruh keputusan yang telah ditetapkan.

---

Main Tasks

1. Theme

Implementasikan tema utama aplikasi.

Palette:

- White (#FFFFFF)
- Yellow (#FFC107)
- Black (#212121)
- Red (#E53935)

Buat Design Token agar mudah digunakan pada seluruh aplikasi.

---

2. Typography

Gunakan font:

Inter

Siapkan style:

- Heading
- Subheading
- Body
- Caption

Seluruh halaman menggunakan typography yang konsisten.

---

3. Layout

Buat layout dasar aplikasi.

Minimal terdiri dari:

- Header
- Main Content
- Bottom Navigation

Belum perlu mengimplementasikan navigasi antar halaman.

---

4. Reusable Components

Buat komponen berikut.

- Primary Button
- Secondary Button
- Card
- Badge
- Chip
- Search Bar
- Bottom Navigation
- Loading Skeleton
- Empty State
- Error State

Seluruh komponen harus reusable.

---

5. App Icons

Gunakan Lucide Icons.

Jangan menggunakan icon pack lain.

---

6. Responsive Layout

Optimalkan tampilan untuk:

- 360 px
- 390 px
- 412 px

Pastikan nyaman digunakan dengan satu tangan.

---

7. Home Placeholder

Perbarui halaman Home.

Tampilkan:

- Header aplikasi
- Search Bar
- Area kosong sebagai placeholder peta
- Bottom Navigation

Belum perlu menampilkan peta asli.

---

Deliverables

Sprint selesai apabila tersedia:

- Theme global
- Typography
- Design Token
- Reusable Components
- Responsive Layout
- Home Placeholder

Seluruh komponen dapat digunakan kembali pada sprint berikutnya.

---

Out of Scope

Jangan mengimplementasikan:

- MapLibre
- OpenStreetMap
- Heatmap
- H3 Grid
- Geolocation
- Bottom Sheet
- Floating Action Button
- Recommendation Card
- Insight Panel
- API
- Backend
- Prediction Engine

Fokus hanya pada UI Foundation.

---

Constraints

- Gunakan Tailwind CSS.
- Gunakan shadcn/ui apabila sesuai.
- Hindari inline style.
- Semua komponen harus reusable.
- Jangan membuat komponen yang hanya dipakai satu halaman.

---

Definition of Done

Sprint dianggap selesai apabila:

- Seluruh komponen dapat dirender tanpa error.
- Responsive pada ukuran layar target.
- Build berhasil.
- Tidak ada TypeScript error.
- Tidak ada lint error.
- Konsisten dengan DESIGN.md.

---

Expected Output

Setelah sprint selesai, Bandung Ride Radar telah memiliki Design System sederhana yang siap digunakan untuk membangun fitur-fitur inti seperti Live Map, Heatmap, Recommendation Card, dan Insight Panel pada sprint berikutnya.
